How to Use

1

Generate your input

Run the SQL query (click the SQL ⓘ button in the Validator tab) in your database client, substituting {campaign_id} with your target program ID(s). Copy the result.

The tool accepts both plain CSV format and MySQL table output (with +---+ borders) — paste either directly.
2

Paste or upload

Switch between Paste and Upload CSV using the toggle in the Input panel. After pasting, a preview renders confirming how many URLs were parsed and what was detected.

3

Configure and run

Adjust settings if needed then click Run. The terminal streams live progress. Results appear in the table as each URL completes — you don't need to wait for the full run.

Identity enrichment runs automatically in the background after crawl completes. Rows update in place as identity data resolves — allow up to 7 minutes.
4

Read results

Use the filter tabs to view by integration type. Click any row to expand a full detail panel showing cookies, redirect chain, identity IDs, and all raw values.

Check results are shown as ✓ (true) / ✗ (false). WARN means the check needs attention. — means not applicable for this integration type.
A WARN on cus_id_present means a CustomerId was passed in an anonymous session — this is a client implementation issue that should be flagged.
5

Export

Use ↓ CSV to download or ⧉ Sheets to copy tab-separated values directly into Google Sheets. Both respect the active filter tab.

About

The Tracking Validator is an internal TSE tool that automates the evaluation of impact.com tracking integrations at scale.

It navigates to each tracking link using a headless Chromium browser, follows the full redirect chain, and validates the tracking implementation against a defined set of checks — covering UTT tag fires, identify calls, CustomProfileId stitching, cookie placement, consent signals, redirect integrity, and identity graph enrichment.

What it checks

  • Click ID present in redirect chain and tracking payloads
  • UTT JavaScript tag detection and load timing
  • Identify call — path, status, CustomProfileId, CustomerId
  • Cookie matching — IR_PI, IR_{campaign_id}, IR_gbd, click ID cookies
  • Shopify Plugin — PageLoad API request, web pixel console signal, consent API
  • Hybrid integration detection
  • Implementation method — GTM, Tealium, Segment, Adobe Launch, or direct
  • Redirect chain with server IP addresses per hop
  • Identity graph enrichment via identity service API
